## Purpose

User-facing React PWA. Accessible at `app.[domain]`.
PWA enabled. Mobile-first.

## Auth

- Email + configured OAuth providers (from project.yaml)
- Self-register: controlled by project.yaml
- Dev login: `DevLogin` component from commons (non-prod only)
- Forgot password / password reset via Supabase (check emailing setup)

## Routes

| Path | Layout | Page | Notes |
|------|--------|------|-------|
| `/login` | AuthLayout | LoginPage | includes DevLogin in non-prod |
| `/` | AppLayout | DashboardPage | demo LLM chat (RemoveMeThisIsOnlyDemoLlmChat) |
| `/settings/profile` | AppLayout | ProfilePage | name, notification prefs |
| `/settings/billing` | AppLayout | BillingPage | hidden if pricing_model: none |
| `/support` | AppLayout | SupportPage | message thread with admin |

## Onboarding flow

`OnboardingModal` from commons shown on dashboard when `onboarding_step < 2`.
Reads + writes `user.settings.onboarding_step` via `PATCH /api/users/me/settings`.
Has "Skip for now" → sets step to 2. Never blocks navigation.

Step 1 (sample): "What brings you here today?"
- [ ] Building a product
- [ ] Testing the template
- [ ] Learning
- [ ] Other

Step 2 (sample): "How did you hear about us?" — free text input.

## Cookie banner

`CookieBanner` from commons renders in `App.tsx` root.
Hidden if `analytics: none`. One-time dismiss stored in `localStorage: cookie_consent`.
PostHog initializes only after consent.

## Billing visibility

Controlled by `config.pricingModel` from commons config. If `none`: hide all billing routes + components. Do not remove code — hide via config check.

```typescript
import { config } from '@any-project-base/commons'
if (config.pricingModel === 'none') return null
```

## PWA

Enabled. `manifest.json` GENERATED by `apply-branding`. Service worker via `vite-plugin-pwa`.
