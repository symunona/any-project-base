Qué incluye

Todo conectado. Nada pendiente de cablear.

Auth

Email, Google, Facebook. Sesiones basadas en cookies vía Supabase. Control de acceso por roles incluido.

Pagos

Créditos, suscripciones o niveles. Stripe + sincronización de webhooks. Cambia de modelo con un solo cambio de configuración.

Portal de administración

Usuarios, tickets de soporte, estadísticas de uso, historial de despliegues. Listo desde el primer día.

Móvil

Expo React Native. Notificaciones push vía Firebase. Deep links. Target web para pruebas.

TypeScript en todas partes

Tipos de extremo a extremo. Esquema de BD → tipos generados → frontend. Sin desfase, sin sorpresas.

Comprobaciones de seguridad

Auditoría de cadena de suministro, guardia de colores CSS, guardia de fetch, umbral de complejidad, detección de desfase de esquema — todo se ejecuta en cada commit.

Multiidioma

HTML estático por idioma. Detección del idioma del navegador. Idioma guardado en cookie + localStorage.

Despliegues

Cloudflare, Vercel o Netlify. Un comando para desplegar a pruebas o producción. Etiquetas de versión y changelogs integrados.

Seguridad primero

Las comprobaciones se ejecutan automáticamente.
Los patrones incorrectos no pueden integrarse.

  • Auditoría de cadena de suministro en cada just install
  • Detección de colores fijos & secretos en CI
  • Umbral de complejidad ciclomática (máx. 8) — impuesto por ESLint
  • Comprobación de desfase de esquema — tipos de BD siempre sincronizados
  • Guardia de fetch — sin llamadas accidentales entre límites de módulos
Nativo para agentes

Cada directorio tiene un
AGENTS.md

Claude, Copilot, Cursor — todos leen archivos de contexto para entender el código. Hemos escrito el manual para ellos. En cada carpeta.

Sin abstracciones incorrectas

Las reglas de arquitectura viven junto al código que gobiernan. Los agentes saben qué NO hacer tanto como qué hacer.

Convenciones consistentes

Un patrón para auth, uno para obtener datos, uno para componentes. Los agentes predicen dónde viven las cosas sin que se les diga.

Umbrales de complejidad

Los verificadores se ejecutan tras cada cambio. Los agentes no pueden subir accidentalmente código que supere el presupuesto de complejidad o importe el módulo incorrecto.

Documentación de arquitectura

Cada decisión importante está documentada en architecture/. Los agentes leen el razonamiento antes de tocar el código.
