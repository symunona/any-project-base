작동 방식

세 가지 명령어로 제로에서 실행까지

01

8가지 질문에 답하세요

just setup을 실행하세요. 프로젝트 이름, 도메인, 가격 모델, 필요한 서비스를 선택합니다. 5분 이내에 완료됩니다. 환경 변수 13개가 자동으로 채워집니다.

$ just setup
→
02

명령어 하나로 모든 것이 시작됩니다

just start를 실행하세요. Supabase가 실행되고, Supabase 키가 .env.local에 자동으로 채워지며, 세 개의 프론트엔드 서버가 모두 시작됩니다. 포트 충돌을 감지하고 자동으로 해결합니다.

$ just start
→
03

중요한 것을 출시하세요

나머지는 이미 모두 준비되어 있습니다. 인증이 작동합니다. 결제가 작동합니다. 어드민 포털이 작동합니다. 당신은 제품을 차별화하는 기능을 작성하면 됩니다.

$ git push → prod
데모까지 걸리는 시간

SaaS 부트스트래핑이 몇 주에서 몇 시간으로 줄어들었습니다

기존 방식
4~6주
any-project-base
약 6시간

측정 기준: "빈 저장소"에서 "인증 + 결제가 작동하는 데모" 완성까지.

포함된 기능

모든 것이 연결되어 있습니다. 직접 연결할 것은 없습니다.

인증

이메일, Google, Facebook. Supabase를 통한 쿠키 기반 세션. 역할 기반 접근 제어가 기본 제공됩니다.

결제

크레딧, 구독, 또는 티어 방식. Stripe + 웹훅 동기화. 설정 하나만 바꾸면 결제 모델이 전환됩니다.

어드민 포털

사용자 관리, 지원 티켓, 사용량 통계, 배포 이력. 첫날부터 바로 사용할 수 있습니다.

모바일

Expo React Native. Firebase를 통한 푸시 알림. 딥 링크. 테스트용 웹 타겟 포함.

TypeScript 전면 적용

엔드투엔드 타입. DB 스키마 → 생성된 타입 → 프론트엔드. 타입 불일치도, 예상치 못한 오류도 없습니다.

보안 검사

공급망 감사, CSS 색상 가드, 페치 가드, 복잡도 게이트, 스키마 드리프트 감지 — 모든 커밋마다 자동으로 실행됩니다.

다국어 지원

로케일별 정적 HTML. 브라우저 언어 자동 감지. 언어 설정이 쿠키와 localStorage에 저장됩니다.

배포

Cloudflare, Vercel, 또는 Netlify. 명령어 하나로 테스트 또는 프로덕션 배포. 릴리즈 태그와 변경 로그가 내장되어 있습니다.

보안 우선

검사가 자동으로 실행됩니다.
잘못된 패턴은 병합될 수 없습니다.

  • just install 실행 시마다 공급망 감사
  • CI에서 하드코딩된 색상 & 시크릿 감지
  • 순환 복잡도 게이트 (최대 8) — ESLint로 강제 적용
  • 스키마 드리프트 검사 — DB 타입이 항상 동기화됨
  • 페치 가드 — 경계를 넘는 API 호출 실수 방지
에이전트 네이티브

모든 디렉터리에
AGENTS.md가 있습니다

Claude, Copilot, Cursor — 모든 AI는 컨텍스트 파일을 읽어 코드베이스를 이해합니다. 우리는 모든 폴더에 그 매뉴얼을 작성해 두었습니다.

잘못된 추상화 없음

아키텍처 규칙이 해당 코드 옆에 위치합니다. 에이전트는 무엇을 해야 하는지만큼 무엇을 하면 안 되는지도 정확히 압니다.

일관된 컨벤션

인증 패턴 하나, 데이터 페칭 패턴 하나, 컴포넌트 패턴 하나. 에이전트는 별도 지시 없이도 어디에 무엇이 있는지 예측합니다.

복잡도 게이트

모든 변경 후 검사기가 실행됩니다. 에이전트가 실수로 복잡도 예산을 초과하거나 잘못된 모듈을 임포트하는 코드를 배포할 수 없습니다.

아키텍처 문서

주요 결정 사항이 architecture/에 문서화되어 있습니다. 에이전트는 코드를 수정하기 전에 그 이유를 먼저 읽습니다.
