## Purpose

Static multilingual landing page. Accessible at `[domain]`.
Used as Android/iOS app intent target via web manifest.
Lightweight — no React, no React Query, no router.

## Differences from frontend.md

- **No React.** Plain HTML + vanilla JS + lightweight Vite build (no JSX).
- **No React Query, no router.** Static files per language.
- **No PWA.** App intent handled via `manifest.json` + meta tags.
- **No auth.** No Supabase client here.
- **No commons components** (React-based). Shared only: `globals.css` color vars.

## Language handling

- Lang selector top-right → redirects to `[lang]/index.html`
- `main.js` detects browser language on first visit → redirects if matching locale available
- Lang pref saved in cookie + localStorage key `lang` (app subdomains read this)
- Separate full HTML file per language — no JS-based i18n at runtime

## YouTube background

`video-bg.js` embeds YouTube video as muted autoplay background behind header.
Video ID configured in landing HTML. Fallback: solid `var(--color-primary)` bg.

## App intent (deep link)

`index.html` includes:
```html
<link rel="manifest" href="/manifest.json">
<meta name="apple-itunes-app" content="app-id=[IOS_APP_ID]">
```

Allows Android/iOS to intercept links to `[domain]` and open the native app.
