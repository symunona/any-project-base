# Frontend Conventions

Applies to: `client-portal/`, `admin-portal/`, `landing/`. Mobile differences in `architecture/mobile.md`.
Each project's `AGENTS.md` references this file and lists only what differs.

---

## Stack

- **Runtime:** pnpm, Vite, React, TypeScript (strict)
- **Styling:** Tailwind CSS + CSS custom properties from `commons/styles/globals.css`
- **Data fetching:** React Query + `@any-project-base/commons/api/client`
- **Routing:** React Router v7
- **Icons:** Lucide React
- **PWA:** Vite PWA plugin (client-portal + admin-portal only)
- **Package versions:** pinned exactly in root `package.json` — no `^` or `~`

---

## Package Management

Use root `package.json` only. No project-level `package.json` for shared deps.

```
package.json          ← all shared deps pinned here
pnpm-workspace.yaml   ← lists all workspace packages
client-portal/
  package.json        ← only project-specific deps not in root
admin-portal/
  package.json        ← only project-specific deps not in root
```

Run installs from root always: `pnpm install` at repo root.
Run GlassWorm + `socket.dev` + `pnpm audit` after every install.

## Directory Structure (per frontend project)

```
[project]/
  index.html
  vite.config.ts
  tsconfig.json          ← extends root tsconfig, strict: true
  public/
    manifest.json        ← GENERATED by apply-branding
    favicon.svg          ← GENERATED (copied from branding/)
    logo.svg             ← GENERATED (copied from branding/)
    logo-small.svg       ← GENERATED (copied from branding/)
  src/
    main.tsx             ← app entry
    App.tsx              ← router setup, providers
    layouts/
      AppLayout.tsx      ← authenticated app shell (header + content)
      AuthLayout.tsx     ← login/register pages shell
    pages/               ← one file per route
    components/          ← project-specific components grouped by module
      [module]/
        [Component].tsx
    hooks/               ← project-specific hooks
    lib/                 ← project-specific utilities
```

Import from commons for shared code. Never duplicate what's in commons.

## Styling Rules

- **Only use CSS custom properties for colors.** Never hardcode hex, rgb(), hsl() anywhere.
- All color vars defined in `commons/styles/globals.css` — import once in `main.tsx`.
- Tailwind: use semantic class names where possible. For colors, use CSS vars via arbitrary values: `bg-[var(--color-primary)]`.
- Dark mode: toggled via `data-theme="dark"` on `<html>`. Persisted in localStorage key `theme`.
- One `globals.css` import in `main.tsx` — no other global CSS files.

```typescript
// main.tsx
import '@any-project-base/commons/styles'
```

Checker: `setup/checks/css_color_check.sh` — greps all `.tsx`/`.css` files for hardcoded colors. CI fail.

---

## Component Rules

- Always search `commons/components/` first before writing a new component.
- Group project-specific components by feature module under `src/components/[module]/`.
- One component per file. File name = component name.
- No inline styles except for truly dynamic values (e.g. computed widths).
- Props typed explicitly — no `any`.

---

## Data Fetching Rules

- **Never use `fetch` directly.** Always use `fetchApi` from commons or the typed Hono client.
- All list endpoints use `usePaged` or `useInfinite` from commons hooks.
- All mutations use React Query `useMutation`.
- No data fetching in components — only in hooks.

```typescript
// correct
function useUsers(params: PageParams) {
  return usePaged('/users', params)  // from commons
}

// wrong — never do this
const res = await fetch('/api/users')
```

ESLint `no-restricted-globals: fetch` enforces this. CI fail if violated.

## Routing

```typescript
// App.tsx
<Routes>
  <Route element={<AuthLayout />}>
    <Route path="/login" element={<LoginPage />} />
  </Route>
  <Route element={<AppLayout />}>
    <Route path="/" element={<DashboardPage />} />
    // ...
  </Route>
</Routes>
```

- One page component per route in `src/pages/`.
- Page components are thin — delegate to feature components.
- Protected routes: `AppLayout` checks auth, redirects to `/login` if not authenticated.

## i18n

All user-facing strings go through `t()`. No raw strings in JSX.

```typescript
import { t, msg } from '@any-project-base/commons/i18n'

// correct
<Button>{t(msg.Common.save)}</Button>

// wrong
<Button>Save</Button>
```

Checker: `setup/checks/i18n_check.sh` — raw string detector in JSX (heuristic grep). Warns on likely missing translations.

---

## Auth

Use `useAuth()` from commons. Never read session directly.

```typescript
const { user, role, isAdmin, loading } = useAuth()
```

Role checks in UI = show/hide only. Never trust frontend role for actual access control — backend enforces.

## Build-time Env Injection

Vite injects at build time via `vite.config.ts`:

```typescript
// vite.config.ts
define: {
  'import.meta.env.VITE_COMMIT_SHA': JSON.stringify(
    execSync('git rev-parse --short HEAD').toString().trim()
  ),
  'import.meta.env.VITE_COMMIT_DATE': JSON.stringify(
    execSync('git log -1 --format=%ci').toString().trim()
  ),
}
```

`CommitWatermark` from commons renders these bottom-right. Shows `dev` if not set.

## PWA (client-portal + admin-portal)

- `vite-plugin-pwa` configured in `vite.config.ts`
- `manifest.json` — GENERATED by `apply-branding`. Never edit manually.
- Service worker: cache-first for static assets, network-first for API calls.
- Mobile-first responsive design. Test at 375px width minimum.

---

## Notifications

Use `Notification` component from commons for in-page toasts. Never use `alert()` or `console.log` for user-facing messages.

```typescript
import { useNotification } from '@any-project-base/commons/hooks'

const { notify } = useNotification()
notify({ type: 'success', message: t(msg.Common.saved) })
notify({ type: 'error', message: t(msg.Errors.generic) })
```

---

## Modals

Use `Modal` from commons. Never use browser `confirm()` or `prompt()`.

```typescript
import { Modal } from '@any-project-base/commons/components'

<Modal open={open} onClose={() => setOpen(false)} title="Confirm deletion">
  ...
</Modal>
```

| `css_color_check.sh` | hardcoded colors | pre-commit + CI |
| `fetch_check.sh` | direct fetch calls | pre-commit + CI |
| `i18n_check.sh` | raw strings in JSX, missing keys | CI |
| `knip` | unused exports/deps | CI |
| `size-limit` | bundle regression | CI |
| GlassWorm + socket.dev | supply chain | on install |
