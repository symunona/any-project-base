## Purpose

Fixed architectural decisions for this project. Read-only for agents during normal work.
Only updated when a new architectural decision is made — requires explicit dev approval.

## Rules

- **Never modify architecture docs during feature work.** These are decisions, not suggestions.
- If implementation requires deviating from an architecture doc: STOP. Flag to dev. Do not proceed.
- If a new pattern emerges that should be standardized: propose addition to dev, do not self-edit.
- `philosophy.md` is the highest-priority doc. When in doubt, read it first.

| Task | Read |
