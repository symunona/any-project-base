## Purpose

Shared code for `client-portal/`, `admin-portal/`, `mobile-app/`.
pnpm workspace package: `@any-project-base/commons`.

**Rules:**
- No app-specific logic here
- No direct `fetch` calls — only in `fetchApi.ts`
- No hardcoded colors — only `var(--color-*)` in CSS
- Never import from `client-portal/`, `admin-portal/`, `mobile-app/` — one-way dependency
- Components must work in all consumers (React web + React Native aware)
- `db.types.ts` — GENERATED, never edit manually

### schemas/user-settings.ts

```typescript
export const UserSettingsSchema = z.object({
  onboarding_step: z.number().int().min(0).default(0),
  dark_mode: z.boolean().default(false),
  notification_settings: z.object({
    email: z.object({
      support_reply:    z.boolean().default(true),
      credit_depletion: z.boolean().default(true),
      payment_failed:   z.boolean().default(true),
    }),
    push: z.object({
      support_reply: z.boolean().default(true),
    }),
  }).default({}),
})
export type UserSettings = z.infer<typeof UserSettingsSchema>
// always parse settings through this schema — never read raw jsonb
```

## Adding a new shared component

1. Check if it already exists here first
2. Build it generic — no app-specific logic
3. Export from `components/index.ts`
4. Works in web (HTML elements) OR explicitly documented as web-only
