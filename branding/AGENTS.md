## Purpose

Source of truth for project visual identity: logos, colors.
Edit here, propagate everywhere via `just setup apply-branding`.
Never edit propagated files directly ÔÇö they will be overwritten.

## Directory Structure


## Rules

- **Edit SVGs here** using any vector editor (Figma, Inkscape, Illustrator). Export as SVG, replace file, run `just setup apply-branding`.
- **Edit colors** in `colors.yaml` only. Never touch `commons/styles/globals.css` directly.
- **Never edit `palette.js`** ÔÇö GENERATED. Re-generate via `just setup branding`.
- **Never edit propagated files** ÔÇö `manifest.json`, `app.json` branding fields, `globals.css` are all overwritten by `apply-branding`.
- SVG files are owned by this directory. All other locations are copies.

## When agent touches this directory

- Branding setup step: agent writes `palette.js` only
- Agent never modifies SVG files after initial generation
- If user requests logo changes: regenerate via `just setup branding` with custom instructions
- If user requests color changes: update `colors.yaml`, run `just setup apply-branding`

## Checker

`setup/checks/branding_check.sh`:
- `colors.yaml` exists and has all 5 required keys
- All propagated CSS vars match `colors.yaml` values
- All manifest `theme_color` values match `colors.yaml` primary
- All SVG destinations exist and are non-empty
- `palette.js` has GENERATED header
- `globals.css` has GENERATED header
