## Purpose

All setup, installation, health check, and platform configuration scripts.
Runs once at project init, re-runnable at any time via `just setup [step]`.

**Script-first principle:** use pure shell wherever possible. Agent calls only when reasoning about project structure is genuinely needed. See `architecture/tooling.md`.

## lib/ui.sh — always source this

```bash
source "$(dirname "$0")/../lib/ui.sh"
```

Provides: `header`, `step`, `info`, `warn`, `success`, `fail`, `arrow`, `skip`, `prompt_input`, `confirm`, `status_row`, `divider`. See `architecture/setup-ux.sh` for full implementations.

Step list display — reprint at top of every step:
- Past + done: `GREEN ✓` white text
- Past + skipped: `YELLOW ↷` dim text
- Current: `CYAN ▶` bold text
- Future: dim gray, no icon

## lib/agent.sh — run_agent()

```bash
source "$(dirname "$0")/../lib/agent.sh"
run_agent "path/to/prompt.md"
# reads agent + agent_flags from project.yaml
# if agent=echo: prints prompt, no call (dry run)
```

Always verify agent output before next step. Never chain agent calls without checking result.

## lib/yaml.sh — read_yaml()

```bash
source "$(dirname "$0")/../lib/yaml.sh"
PRICING=$(read_yaml pricing_model)
AGENT=$(read_yaml agent)
```

## Platform setup script pattern

Every `[platform]_setup.sh` follows identical structure:

```bash
#!/bin/bash
source "$(dirname "$0")/../lib/ui.sh"

header "PLATFORM NAME"
info "What this platform does."
warn "Without this: what breaks."
echo ""

# step-by-step instructions
arrow "Go to https://platform.com/register"
arrow "Click New Project → copy API key"
echo ""

# interactive key input
prompt_input "Paste API key" API_KEY || { skip "Skipping platform."; exit 0; }

# validate immediately
if ! validate_api_key "$API_KEY"; then
  fail "Invalid key. Try again or press s to skip."
  # retry or skip
fi

# write to .env.local
echo "PLATFORM_API_KEY=$API_KEY" >> .env.local
success "Platform configured."
```

## Check script pattern

Every `[platform]_check.sh` outputs machine-readable status for health table:

```bash
#!/bin/bash
# outputs: OK | SKIP | FAIL [reason]
source "$(dirname "$0")/../lib/ui.sh"

if [ -z "$PLATFORM_API_KEY" ]; then
  echo "SKIP missing API key"
  exit 1
fi

if ! curl -sf -H "Authorization: Bearer $PLATFORM_API_KEY" https://api.platform.com/ping; then
  echo "FAIL invalid or expired key"
  exit 1
fi

echo "OK"
```

## Rules for writing setup scripts

- Source `lib/ui.sh` at top of every script
- Every prompt shows `(s=skip, q=quit)`
- Every step writes result to `setup/.install-state.json`
- Validate API keys immediately after input — never write invalid keys to .env
- Agent prompts in `prompts/` are self-contained — full context, exact file paths, no guesswork
- Verify agent output before proceeding to next step
- Check scripts output exactly: `OK`, `SKIP [reason]`, or `FAIL [reason]`
